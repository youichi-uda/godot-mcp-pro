# Godot MCP Pro

Премиальный MCP (Model Context Protocol) сервер для разработки игр на Godot с помощью ИИ. Подключает ИИ-ассистентов, таких как Claude, напрямую к вашему редактору Godot с **163 мощными инструментами**.

## Архитектура

```
AI Assistant ←—stdio/MCP—→ Node.js Server ←—WebSocket:6505—→ Godot Editor Plugin
```

- **Реальное время**: WebSocket-соединение обеспечивает мгновенную обратную связь, без опроса файлов
- **Интеграция с редактором**: Полный доступ к API редактора Godot, системе UndoRedo и дереву сцен
- **JSON-RPC 2.0**: Стандартный протокол с корректными кодами ошибок и подсказками

## Быстрый старт

### 1. Установка плагина Godot

Скопируйте папку `addons/godot_mcp/` в директорию `addons/` вашего проекта Godot.

Включите плагин: **Проект → Настройки проекта → Плагины → Godot MCP Pro → Включить**

### 2. Установка MCP-сервера

> **Примечание**: Директория `server/` включена только в **полный пакет** (платный).
> Этот GitHub-репозиторий содержит только **аддон (плагин)**.
> Приобретите полный пакет на [godot-mcp.abyo.net](https://godot-mcp.abyo.net/), чтобы получить сервер.

```bash
cd server
npm install
npm run build
```

### 3. Настройка Claude Code

Добавьте в ваш `.mcp.json`:

```json
{
  "mcpServers": {
    "godot-mcp-pro": {
      "command": "node",
      "args": ["D:/dev/godot-mcp-pro/server/build/index.js"],
      "env": {
        "GODOT_MCP_PORT": "6505"
      }
    }
  }
}
```

### 4. Автоматическое одобрение разрешений (Рекомендуется)

Claude Code запрашивает разрешение при каждом вызове инструмента. Чтобы пропустить эти запросы, скопируйте один из включённых пресетов разрешений в настройки Claude Code:

**Вариант A: Консервативный** (по умолчанию — блокирует деструктивные инструменты)

```bash
cp settings.local.json ~/.claude/settings.local.json
```

Автоматически разрешает 152 из 163 инструментов. Следующие 11 инструментов по-прежнему требуют ручного одобрения:

| Заблокированный инструмент | Причина |
|---|---|
| `delete_node` | Удаляет узел из сцены |
| `delete_scene` | Удаляет файл сцены с диска |
| `remove_animation` | Удаляет анимацию |
| `remove_autoload` | Удаляет синглтон autoload |
| `remove_state_machine_state` | Удаляет состояние из state machine |
| `remove_state_machine_transition` | Удаляет переход state machine |
| `execute_editor_script` | Выполняет произвольный код в редакторе |
| `execute_game_script` | Выполняет произвольный код в запущенной игре |
| `export_project` | Запускает экспорт проекта |
| `tilemap_clear` | Очищает все ячейки TileMapLayer |

**Вариант B: Разрешительный** (разрешает всё, блокирует опасные команды)

```bash
cp settings.local.permissive.json ~/.claude/settings.local.json
```

Разрешает все 163 инструмента и все Bash-команды. Явно блокирует деструктивные shell-команды (`rm -rf`, `git push --force`, `git reset --hard` и т.д.) и те же деструктивные MCP-инструменты, перечисленные выше.

> **Примечание**: Если у вас уже есть `settings.local.json`, вручную объедините секцию `permissions` вместо перезаписи.

### 5. Lite-режим (Опционально)

Если ваш MCP-клиент имеет ограничение на количество инструментов (например, Windsurf: 100, Cursor: ~40), используйте Lite-режим, который регистрирует 76 основных инструментов вместо 162:

```json
{
  "mcpServers": {
    "godot-mcp-pro": {
      "command": "node",
      "args": ["D:/dev/godot-mcp-pro/server/build/index.js", "--lite"]
    }
  }
}
```

Lite-режим включает: инструменты project, scene, node, script, editor, input, runtime и input_map.

### 6. Использование

Откройте проект Godot с включённым плагином и используйте Claude Code для взаимодействия с редактором.

## Ключевые возможности

- **Интеграция UndoRedo**: Все операции с узлами/свойствами поддерживают Ctrl+Z
- **Умный парсинг типов**: `"Vector2(100, 200)"`, `"#ff0000"`, `"Color(1,0,0)"` конвертируются автоматически
- **Автопереподключение**: Переподключение с экспоненциальным откатом (1с → 2с → 4с ... → 60с макс)
- **Heartbeat**: Ping/pong каждые 10с поддерживает соединение активным
- **Полезные ошибки**: Ответы об ошибках включают подсказки для следующих шагов

### Архитектурные преимущества

| Аспект | Godot MCP Pro | Типичный конкурент |
|--------|--------------|-------------------|
| **Протокол** | JSON-RPC 2.0 (стандартный, расширяемый) | Кастомный JSON или CLI |
| **Соединение** | Постоянный WebSocket с heartbeat | Подпроцесс на команду или сырой TCP |
| **Надёжность** | Автопереподключение с экспоненциальным откатом (1с→60с) | Требуется ручное переподключение |
| **Типобезопасность** | Умный парсинг типов (Vector2, Color, Rect2, hex-цвета) | Только строки или ограниченные типы |
| **Обработка ошибок** | Структурированные ошибки с кодами + подсказками | Общие сообщения об ошибках |
| **Поддержка отмены** | Все изменения через систему UndoRedo | Прямые модификации (без отмены) |
| **Управление портами** | Автосканирование портов 6505-6509 | Фиксированный порт, возможны конфликты |
